# 三个服务中生成空图片的情况分析

## 1. misty-service (MISTY Contributions)

### 可能出现空图片的情况：

1. **R脚本执行失败但文件已创建**
   - 当 `plot_stacked_bar()` 函数调用R脚本时，如果R脚本在创建PNG文件后失败（例如R包缺失、内存不足等），可能会留下一个空文件或损坏的文件
   - 代码只检查 `os.path.exists(final_path)`，不检查文件大小或有效性
   - **位置**: `main.py:300-308`

2. **数据为空但未提前检查**
   - 虽然代码在 `df.empty` 时会生成占位图，但如果数据在传递给R脚本后变为空（例如列名不匹配），R脚本可能生成空图片
   - **位置**: `main.py:275-281`

3. **列名不匹配导致数据为空**
   - 如果 `target_gene` 列不存在，代码会尝试重置索引，但如果重置后仍没有正确的列，传递给R脚本的数据可能为空
   - **位置**: `main.py:283-286`

## 2. giotto-cpdb-service (Giotto-CPDB Heatmap)

### 可能出现空图片的情况：

1. **pivot_table结果为空**
   - 当 `df` 不为空，但 `pivot_table()` 操作后结果为空时（例如所有sender或receiver都相同），会传递给R脚本一个空的DataFrame
   - `plot_heatmap()` 函数虽然检查 `df.empty`，但这是在pivot之前检查的
   - **位置**: `main.py:353-368`

2. **R脚本执行失败但文件已创建**
   - 同misty-service，R脚本可能在创建文件后失败，留下空文件
   - 代码只检查文件是否存在，不检查文件大小
   - **位置**: `main.py:372-380`

3. **所有zscore值相同**
   - 如果所有互动的zscore值都相同（例如都是0），R脚本可能生成一个几乎空白的图片（只有颜色条，没有实际数据）

## 3. spaotsc-service (SpaOTsc Network)

### 可能出现空图片的情况：

1. **edges_df为空但centroids不为空**
   - 当 `df.head(top_edges)` 为空时，`edges_df` 会是空的DataFrame
   - 虽然 `centroids` 不为空，但如果R脚本无法处理只有节点没有边的情况，可能生成空图片
   - **位置**: `main.py:318-331`

2. **R脚本执行失败但文件已创建**
   - 同其他服务，R脚本可能在创建文件后失败
   - **位置**: `main.py:344-352`

3. **sender或receiver在centroids中不存在**
   - 如果 `df` 中的 `sender` 或 `receiver` 在 `centroids` 中不存在，`edges_plot` 会抛出KeyError，但如果在try块中捕获，可能导致部分数据丢失，最终生成不完整的图片

## 共同问题

### 1. 文件存在性检查不充分
所有三个服务都只检查文件是否存在（`os.path.exists()`），不检查：
- 文件大小是否为0
- 文件是否为有效的PNG格式
- 文件是否损坏

### 2. R脚本错误处理不完善
- R脚本可能在创建文件后失败，留下空文件
- `_run_rscript()` 函数只返回True/False，不检查输出文件的有效性

### 3. 数据验证不足
- 在数据转换（pivot、过滤等）后，没有再次验证数据是否为空
- 传递给R脚本的数据可能在实际绘图时变为空

## 建议的修复方案

1. **添加文件有效性检查**
   ```python
   def _is_valid_png(file_path: str) -> bool:
       """检查PNG文件是否有效且非空"""
       if not os.path.exists(file_path):
           return False
       if os.path.getsize(file_path) == 0:
           return False
       try:
           from PIL import Image
           img = Image.open(file_path)
           img.verify()
           return True
       except:
           return False
   ```

2. **在数据转换后添加验证**
   - 在 `pivot_table()` 后检查结果是否为空
   - 在创建 `edges_df` 后检查是否为空

3. **改进R脚本错误处理**
   - 在R脚本执行失败时，删除可能创建的空文件
   - 检查R脚本的输出文件是否有效

4. **添加更详细的日志**
   - 记录数据转换后的形状和大小
   - 记录R脚本的执行状态和输出
