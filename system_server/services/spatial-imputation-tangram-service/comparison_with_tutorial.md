## 四、训练基因选择策略对比

### 教程方法
1. 使用 `rank_genes_groups` 选择标记基因
2. 从每个细胞亚类选择前100个标记基因
3. 去重后得到 ~1,401 个候选基因
4. 经过 `pp_adatas` 过滤后得到 **1,280 个训练基因**
5. **优势**: 使用差异表达基因，更有生物学意义
6. **特点**: 基因数量少但质量高

### 当前方法
1. 使用所有重叠基因作为训练基因
2. 共 **18,000 个训练基因**
3. **优势**: 使用更多基因，可能捕获更多信息
4. **特点**: 基因数量多，包含非差异表达基因

## 六、建议与结论

### 当前结果是否与教程相近？

**结论**: **在核心功能上相近，但在策略上有差异**

1. **功能层面** ✅
   - 都成功完成了 Tangram 映射
   - 都生成了插补的空间数据
   - 都使用了相同的核心算法

2. **策略层面** ⚠️
   - 训练基因选择策略不同（所有重叠基因 vs 标记基因）
   - 训练轮数较少（250 vs 500）
   - 数据规模更大（可能影响训练效果）

3. **质量评估** 📊
   - 当前结果生成了有效的输出文件
   - 映射分数分布合理（1.31-9.30）
   - 需要进一步验证插补质量（如与原始空间数据的相关性）

### 建议改进方向

1. **训练基因选择**
   - 考虑使用差异表达基因作为训练基因（如教程方法）
   - 可以添加参数让用户选择：`all_overlapping` 或 `marker_genes`

2. **训练轮数**
   - 可以增加到 500 epochs（如教程）
   - 或者添加早停机制（当分数不再提升时停止）

3. **质量验证**
   - 添加训练基因的相似度分数（如教程的 Score）
   - 添加测试基因的预测质量评估（AUC曲线等）

4. **参数优化**
   - 对于大规模数据，可能需要调整学习率和正则化参数
   - 考虑根据数据规模自动调整训练轮数
