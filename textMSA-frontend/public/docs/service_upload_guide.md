# STAnalyzer 服务接入与上传指南

在 STAnalyzer 平台中，你可以灵活地将自己的生信分析脚本、模型算法等封装并接入为可共享的「服务(Service)」。这不仅能方便自己日后在平台上直接运行分析，也可以将其公开（如果在平台中允许公开）以供其他研究者使用。

本文档将指引你如何将一个算法接入平台，并在「服务管理」页面中完成服务的上传与注册。

---

## 一、服务的架构与接入原理

STAnalyzer 使用了轻量级的分布式微服务架构。要接入服务，你的算法需要满足以下基本条件：
1. **提供 HTTP(S) 接口**：你的算法或脚本需要被封装在一个 Web API 中（例如使用 Python 的 FastAPI / Flask 等）。平台会通过 HTTP 请求将数据和参数发送给你的服务。
2. **特定的输入与输出**：平台允许动态设定算法执行所需要的文件(如 `.h5ad`, 图像文件等)、输入参数和返回的结果。

## 二、服务上传（注册）步骤

当你的后端 API 准备好后，就可以在 STAnalyzer 前端的「管理」-「服务管理」中开始注册你的服务了。

### 1. 开始创建新服务
登录后，进入左侧导航栏的 **[服务管理 (Service Management)]**，点击右上角的 **[新建服务 (Create Service)]** 开始配置。

### 2. 填写基础信息 (Basic Information)
这是服务的基本身份信息：
* **名称 (Name)**：你的分析服务名称，必须填写。例如“空间转录组 PASTE 对齐”。
* **描述 (Description)**：简单描述这个服务能做什么，适合哪些数据。
* **版本 (Version)**：默认如 1.0.0。
* **权限 (Visibility)**：可以选择设为`私有(Private)`或`公开(Public)`。

### 3. 配置服务通信地址
这部分定义了平台如何找到你的算法服务器。
* **基础服务地址 (Base URL)**：你的后端服务器地址。例如 `http://sdu-205:8050`（局域网）或 `https://api.yourlab.com`。
* **服务接口后缀 (Service API Suffix)**：用于发送计算请求的路径。例如你的后端接口是 `/api/v1/align`，则填此路径。请求时将使用 POST 方法。
* **下载接口后缀 (Download API Suffix) (可选)**：如果你的服务处理完毕后产生的新文件也在同一服务器供下载，可以填写下载前缀，如 `/api/v1/files/{file_id}/download`。

### 4. 定义输入参数 (Input Parameters)
如果你的算法执行时需要调节参数（如阈值、聚类数量等），在此配置：
* 点击 **[添加参数 (Add Parameter)]**
* 填写变量名 (Name)、类型 (Boolean / Continuous / Discrete / Enum / String)。
* 你可以为参数设置「默认值」与「限制条件」(如最大最小值)，有助于前端自动校验。

### 5. 定义所需文件 (Required Files - Accepted Files)
多数服务需要用户传入至少一个文件（如矩阵数据、图像等）。
* **文件名 (Filename)**：通常在向你发送请求时分配的字段标识，如 `slice1`。
* **文件类型ID列表 (File Type IDs)**：指定支持的平台文件类型（如 `h5ad`, `csv` 等），确保传入的文件符合要求。

### 6. 配置输出结果 (Output Configuration)
执行成功后你的服务将返回什么？
* **输出文件 (File)**：服务产生的新数据文件。需要指定文件返回的扩展类型（如预处理后产生新的 `h5ad`）。
* **输出文本 (Text)**：执行日志、统计指标准确率等纯文本反馈。

### 7. 使用快捷 JSON / YAML 配置
如果你觉得手动填写麻烦，可以通过右侧面板 **导入 JSON/YAML 配置**。你可以点击 `[下载示例 JSON]` 获取完整模板结构，修改后点击 `[上传文件]` ，表格会被立刻自动填充！

### 8. 保存与测试
检查所有带有红星必填的内容，点击右上角 **[保存 (Save)]**。创建成功后，你可以在服务列表中看到它，并且随时可以点击「执行 (Execute)」或者前往分析界面测试你的服务了。
